# PageDye

官网：[pagedye.pages.dev](https://pagedye.pages.dev)

给任意网站设置自定义背景——纯色、渐变、图片或动态特效壁纸，按站点独立保存。配置完全存储在本地、无遥测；只有在你主动使用远程图片或 URL 背景时，浏览器才会访问对应地址。

支持 **Chrome / Edge / Brave** 等 Chromium 内核浏览器，以及 **Firefox**（含 Firefox for Android）。手机 / 平板可以用精简的 **PageDye Lite** 用户脚本（Android 上的 Edge / Firefox、iOS/iPadOS 上的 Safari）。

## 功能特性

- **多种背景类型**：纯色、线性/径向渐变、图片（本地文件或 URL），或 16 种动态特效壁纸（Matrix 雨、粒子、极光、雪花、打字机等）
- **自定义动效 API**（v0.6，仅扩展版）：在设置页里编写自己的 Canvas 动效，实时预览，和内置效果共用同一套密度/速度/配色控件，支持单个动效导出/导入分享，详见[自定义动效 API 文档](https://pagedye.pages.dev/custom-effects.html)
- **磨砂玻璃**：为指定卡片/容器元素加毛玻璃背景，透出下层壁纸，颜色、模糊度独立可调
- **昼夜双态联动**：跟随系统 `prefers-color-scheme` 自动切换浅色/深色壁纸
- **幻灯轮换**：多张壁纸按间隔（每次打开 / 15 分钟 / 30 分钟 / 1 小时 / 1 天）自动轮换，支持随机顺序
- **高级图片滤镜**：亮度、对比度、灰度、色相旋转、反色，实时预览
- **背景选择器（元素拾取器）**：当站点自身 CSS 遮挡整页背景时，直接把背景应用到指定元素
- **深度兼容模式**（v0.6.3，v0.7 起还能处理"多张小卡片拼满全屏"的情况）：应对像 Google 移动端这类被不透明容器（或大量不透明小卡片）铺满整个视口的顽固网站，自动检测并清除这些遮挡层，无需手动逐个排查；面板里放在最前面，方便第一时间发现
- **自定义 CSS**：为任意站点注入自定义样式
- **配置预设**：一键保存当前站点的完整设置，内置浅色极光、深色极光、日间和夜间预设，并提供缩略图
- **站点分组与批量操作**：把预设应用到多个站点、复制站点配置、维护工作/娱乐等分组，只导入或导出选中站点
- **设置面板**：站点管理、带版本化格式校验的备份/还原（含本地图片）、一键清空所有站点配置

## 安全与隐私说明

- **备份导入**：当前使用 schema v3，备份站点配置、URL 规则、全站默认、自定义动效、自定义配色、配置预设和站点分组，并继续兼容 schema v1、v2 与旧版原始 JSON。导入不会覆盖调试开关、总开关、界面主题等保留项；选中站点导入会先显示文件内站点供勾选。
- **URL 规则（v0.8.1）**：可按 hostname、精确网址、路径前缀或通配子域名应用独立背景，也可排除页面。规则从上到下匹配，支持拖动排序和临时禁用；原有 hostname 配置作为未命中规则时的回退。
- **自定义 Canvas 动效**：动效代码会在独立的扩展沙箱 iframe 中执行；沙箱没有扩展 API 权限，并通过 CSP 禁止联网，宿主网页的 CSP 也不会再阻止动效编译。导入确认会先于任何代码验证/执行，且文件、代码和动效数量均有限制；仍应只导入可信文件。
- **URL 动效背景**：仅接受 HTTPS 地址（`localhost`/回环地址可使用 HTTP），并在不授予同源、弹窗和顶层导航权限的沙箱 iframe 中运行。部分网站会因为 CSP 或 `X-Frame-Options` 无法嵌入，这是预期的浏览器安全行为。
- **调试模式**：调试脚本默认不会注入网页，只在用户主动打开调试模式后按需加载。网络调试可在本地读取请求/响应摘要用于排错，数据不会上传；使用完建议关闭调试模式。
- **扩展权限**：`storage`/`unlimitedStorage` 用于保存配置和本地图片，`scripting` 与 `<all_urls>` 用于把用户选择的背景应用到网站。v0.8.0 已移除不必要的 `activeTab` 权限。

## 安装

最简单的方式是直接访问官网 **[pagedye.pages.dev](https://pagedye.pages.dev)**，桌面端下载扩展包、手机 / 平板端一键安装 PageDye Lite，页面上都有对应的按钮和分步说明。

也可以按下面的方式手动安装。

扩展尚未上架各浏览器商店，目前需要手动加载已解压的扩展包。

前往 [**Releases**](../../releases) 下载最新的正式版本 `.zip`，或前往 [**Actions**](../../actions) → 选择最近一次成功的运行 → 下载页面底部的 artifact，然后解压。

### Chrome / Edge / Brave 等 Chromium 内核浏览器

1. 地址栏输入 `chrome://extensions`（Edge 为 `edge://extensions`）。
2. 打开右上角的**开发者模式**。
3. 点击**加载已解压的扩展程序**，选择解压后的文件夹。

### Firefox（需 140 及以上版本；Firefox for Android 需 142 及以上）

扩展未经 Mozilla 签名，只能以“临时扩展”方式加载，**浏览器重启后需要重新加载一次**：

1. 地址栏输入 `about:debugging#/runtime/this-firefox`。
2. 点击**临时载入附加组件…**。
3. 选择解压后文件夹内的 `manifest.json`。

## 手机 / 平板安装（PageDye Lite）

手机浏览器大多没有完整扩展开发者模式，PageDye 额外提供了一个精简的 [**PageDye Lite**](userscript/pagedye.user.js) 用户脚本（UserScript）版本，通过脚本管理器运行，核心渲染逻辑和正式扩展版共享，功能是完整版的子集（多站点仪表盘、跨标签页实时联动、右键菜单入口等在脚本内说明里列了取舍）。

推荐直接在手机浏览器打开 **[pagedye.pages.dev](https://pagedye.pages.dev)**，点击**安装 PageDye Lite** 按钮一键安装（前提是已经装好脚本管理器，见下）；也可以按下面的步骤手动安装。

### Android — Edge / Firefox

1. 安装脚本管理器 [Tampermonkey](https://www.tampermonkey.net/)：Edge、Firefox 移动版现在都已支持在扩展/附加组件商店里直接搜索安装 Tampermonkey。
2. 打开 Tampermonkey，新建一个脚本。
3. 打开 [`userscript/pagedye.user.js`](userscript/pagedye.user.js) 文件，全选复制其内容，粘贴进去覆盖默认模板，保存。

> Chrome 移动版官方不支持任何扩展，无法安装 Tampermonkey；需要用 Chromium 内核浏览器的话可以换 Edge、Firefox，或 Kiwi Browser 等支持扩展的第三方分支。

### iOS / iPadOS — Safari

1. App Store 搜索安装 **Userscripts**（作者 Quoid，免费开源）。
2. 设置 → Safari → 扩展 → 启用 **Userscripts**，并允许在所有网站上运行。
3. 打开 [`userscript/pagedye.user.js`](userscript/pagedye.user.js) 复制内容，在 Userscripts App 里新建脚本并粘贴保存。

安装完成后，打开任意网站，右下角会出现一个悬浮按钮，点开即为设置面板；按钮的颜色、大小、图标、拖动、贴边隐藏都可以在面板的**高级设置**标签页里自定义。

## 使用

1. 访问任意网站，点击 PageDye 图标。
2. 选择背景类型（纯色 / 渐变 / 图片 / 特效），调整不透明度、模糊度等参数，保存即可生效。

对于大多数网站，到这里就完成了。若网站自身的 CSS 遮挡了整页背景，请使用下方的背景选择器。

## 背景选择器（元素拾取器）

部分网站会用自己的不透明元素覆盖页面背景，导致整页覆盖层不可见。背景选择器允许你将背景直接应用到任意元素上。

打开**高级设置 → 背景选择器**，先配置好颜色或图片，再点击**拾取**。弹窗关闭后，页面上会出现一个蓝色高亮框跟随鼠标移动，同时显示当前元素的 CSS 选择器。**点击**即可将背景应用到该元素，按 **Esc** 取消。

背景通过 `!important` 直接作用于目标元素（图片模式使用 `::before` 层，确保不透明度和模糊效果不影响元素内的文字和子元素）。效果立即生效，刷新页面后依然保留。

也可以跳过拾取，直接手动输入 CSS 选择器（如 `#app`、`.layout-bg`）。

## 深度兼容模式

有些网站（典型例子是 Google 移动端搜索结果页）无论怎么设置都看不到壁纸：整页覆盖层背后总有另一层不透明背景挡着；背景选择器又只能指定一个元素，指哪打哪，遮不住上面还叠着的其他不透明层。深度兼容模式就是为这类"顽固网站"准备的——开启后不需要再逐个排查是哪个元素挡住了背景。

因为它是最常见的"为什么背景没生效"问题的解法，面板里把它放在了**第一位、默认展开**，不再和背景选择器、自定义 CSS 一起藏在"高级设置"里。

### 工作原理

打开开关后，PageDye 会在当前视口上按 6×8 = 48 个采样点用 `elementsFromPoint` 探测每个点上的元素堆栈，用两条规则识别"挡住背景的不透明层"：

1. **单一大容器**：某个元素自身的不透明背景，覆盖了当前视口面积的 50% 以上——对应"好几层不透明容器叠在一起"的经典情况。
2. **马赛克式小卡片**（v0.7 新增）：像 Google 移动端搜索结果那样，页面本身的包裹层是透明的，真正挡住背景的是一大堆各自不到 50% 视口面积、但拼起来铺满全屏的小卡片（每条搜索结果一张）。这种情况下不存在符合规则 1 的"大家伙"，所以 PageDye 额外统计 48 个采样点里有多少个点的最前景元素本身是不透明的——如果超过 60%，就把这些前景元素也一并清除背景。

命中规则的元素会被强制加上 `background-color: transparent !important` 和 `background-image: none !important`（内联样式，优先级足以压过网站自己的样式表），原始的 `style` 属性会先备份，条件不再满足时（比如滚动出视口、被网站自己改了背景、或者关闭了开关）立即还原。

扫描不是只跑一次：页面滚动、窗口尺寸变化、DOM 有新增/删除节点都会触发重新扫描（400ms 防抖），另外每 3 秒还有一次保底扫描，用来兜住防抖节流漏掉的情况（比如某个已有元素只是尺寸变了但没有增删节点）。

### 排除选择器

这个模式偏"暴力"，个别情况下可能误清除某个依赖背景色做对比度的元素——不管是规则 1 的大容器，还是规则 2 揪出来的小卡片，只要看起来不该被清空，都可以在**排除选择器**里填入它的 CSS 选择器（如 `.modal, [role=dialog]`），把它排除在扫描范围之外，`:closest()` 语义——填的选择器命中该元素本身或其任意祖先都算排除。

### 什么时候该用

它是**按站点单独开启**的选项，不建议全局默认打开：常规网站用整页覆盖或背景选择器就够了，只有确认背景怎么设都被某个说不清是谁的不透明层挡住时，才需要打开这个开关排查。扩展版和 PageDye Lite 都支持。

## 设置面板

点击弹窗底部的**设置**（或右键扩展图标 → 选项）打开完整的设置面板，可以：

- 查看和编辑所有已配置站点的背景设置
- 导出 / 导入备份（JSON，含 base64 编码的本地图片）
- 一键清空所有站点配置
